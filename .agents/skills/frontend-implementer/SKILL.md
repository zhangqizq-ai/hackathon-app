# Frontend Implementer

Use this skill when modifying the app UI or frontend behavior.

## Project constraints

- Use plain HTML, CSS, and JavaScript.
- Do not add a framework unless explicitly approved.
- Keep GitHub Pages compatibility.
- Prefer small, reviewable changes.

## Process

1. Inspect index.html, style.css, and script.js.
2. Explain the planned change before editing.
3. Make the smallest necessary change.
4. Preserve existing behavior unless asked to change it.
5. After editing, summarize the diff and provide test steps.

## Definition of done

- The app works by opening index.html.
- No console errors.
- UI remains usable on mobile.
- Changes are easy to commit.
