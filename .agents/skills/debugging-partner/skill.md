# Debugging Partner

Use this skill when the user reports an error, broken UI, unexpected behavior, or failed command.

## Process

1. Ask for the exact error only if it is missing.
2. Inspect the relevant file or command.
3. Identify the likely root cause.
4. Suggest the smallest safe fix.
5. Explain how to verify the fix.

## Rules

- Do not rewrite unrelated code.
- Do not hide uncertainty.
- Prefer one fix at a time.
- If multiple causes are possible, rank them.
