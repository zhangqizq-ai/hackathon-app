# GitHub README Writer

Use this skill before publishing or submitting the hackathon project.

## Goal

Create a clear README for judges, teammates, and future contributors.

## README sections

Include:

1. Project name
2. One-sentence summary
3. Problem statement
4. Target users
5. Features
6. Demo link
7. Screenshots placeholder
8. Tech stack
9. How to run locally
10. How it works
11. Future improvements

## Style

- Clear and concise.
- Avoid hype.
- Make the project easy to understand in 30 seconds.
