# Hackathon MVP Slicer

Use this skill when the user has a product idea that may be too large for a hackathon.

## Goal

Turn a broad idea into a small, demoable MVP.

## Process

1. Identify the target user.
2. Identify the painful problem.
3. Define the smallest useful demo.
4. Split the work into:
   - Must-have
   - Nice-to-have
   - Skip for hackathon
5. Recommend a 2-hour, 4-hour, and 1-day version.

## Output format

Return:

- One-sentence product description
- Target user
- MVP feature list
- Out-of-scope list
- Suggested implementation order
- Demo story
