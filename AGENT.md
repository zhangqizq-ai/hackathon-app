# Project Instructions for Codex

This is a hackathon demo project.

## Tech stack

- Plain HTML
- Plain CSS
- Plain JavaScript
- GitHub Pages deployment
- No build step unless explicitly approved

## Coding rules

- Keep changes small and easy to review.
- Do not introduce frameworks unless asked.
- Preserve compatibility with GitHub Pages.
- Prefer simple readable code over clever code.
- After editing, explain which files changed and why.

## Workflow

Before making changes:
1. Inspect the current files.
2. Summarize the intended change.
3. Ask for approval if the change touches many files.

After making changes:
1. Explain the diff.
2. Suggest how to test locally.
3. Suggest a commit message.
